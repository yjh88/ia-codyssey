# 국어 학습 AI 시스템 개발 보고서

## 1. 프로젝트 개요

본 프로젝트의 목표는 중학생을 대상으로 국어 문법 학습을 돕는 AI 시스템을 설계하는 것이다.  
특히 문장 성분, 품사, 띄어쓰기, 어문 규정처럼 학생들이 자주 헷갈리는 영역에서 **정확하고 친절한 설명**, **난이도별 문제 생성**, **서술형 해설 제공**이 가능한 AI를 구축하고자 하였다.

```css
[프로젝트명] 중학생 국어 문법 학습 AI

[목표]
중학생이 국어 문법 개념을 쉽게 이해하고,
문제 풀이와 해설을 통해 스스로 학습할 수 있도록 돕는 AI 시스템 개발

[핵심 기능]
- 문법 개념 설명
- 난이도별 문제 생성
- 객관식/주관식/서술형 변환
- 오답 포인트 정리
- 어문 규정 관련 환각 방지
- 중학생 눈높이에 맞춘 대화 유지
```

## 2. 모델 비교 및 선정 결과

여러 모델을 비교한 결과, 학습자 수준 분류 및 문제 난이도 예측 측면에서는 **Random Forest 모델**이 가장 안정적인 성능을 보였다.  
정확도는 약 **0.90**으로 나타났으며, 단순 모델보다 다양한 입력 특성을 반영하는 데 유리했다.

```css
[모델 비교 축]

1) 정확성
- 문법 개념, 품사, 문장 성분 판단의 정확도

2) 한국어 자연스러움
- 중학생이 이해하기 쉬운 표현 사용 여부

3) 긴 문맥 유지
- 10턴 이상 대화에서 앞선 조건을 유지하는 능력

4) 형식 준수
- 문제 / 정답 / 해설 형식 유지 여부

5) 난이도 조절
- 하, 중, 상 난이도에 맞는 문제 생성 가능 여부

6) 안정성
- 조건 변경 시에도 출력 품질을 유지하는지

7) 환각 방지
- 모르는 내용에 대해 지어내지 않고 확인을 권하는지
```

```css
[선정 결과]

최종 선정 모델:
Random Forest 기반 학습자/난이도 분류 모델 + LLM 기반 국어 학습 응답 생성 구조

선정 근거:
1) Random Forest는 약 0.90의 정확도를 보여 분류 성능이 우수했다.
2) 다양한 입력 특성, 예를 들어 학년, 단원, 난이도, 오답 유형을 함께 반영하기 적합했다.
3) LLM과 결합할 경우 학생 수준에 맞춘 설명과 문제 생성을 유연하게 수행할 수 있었다.

결론:
국어 학습 AI에는 난이도와 학습자 상태를 판단하는 분류 모델,
그리고 자연어 설명과 문제 생성을 담당하는 LLM을 함께 사용하는 구조가 적합하다.
```

## 3. 타겟 사용자 및 입력 템플릿

이 시스템의 주요 사용자는 **중학생**, 특히 문법 개념을 어려워하거나 내신 대비가 필요한 학생이다.  
입력 템플릿은 AI가 학습자의 수준과 요구를 정확히 파악할 수 있도록 9개 항목으로 구성하였다.

```css
[국어 학습 과업 입력 템플릿]

[학년]
예: 중3

[단원]
예: 문장 성분

[세부 주제]
예: 주어, 서술어, 목적어, 보어 구분

[학습자 수준]
예: 기초 / 중간 / 상위권

[원하는 결과]
예: 개념 설명 / 문제 생성 / 오답 정리 / 서술형 답안 첨삭

[문제 형식]
예: 객관식 / 단답형 / 서술형 / 혼합형

[난이도]
예: 하 / 중 / 상

[톤]
예: 친절한 중등 국어 선생님 말투

[주의사항]
예: 어려운 용어는 쉽게 풀어서 설명하고, 모르는 규정은 단정하지 말 것
```

## 4. 페르소나 및 시스템 프롬프트 요약

AI의 페르소나는 **대치동 중등 국어 전문 강사**로 설정하였다.  
전문성은 유지하되, 학생에게 지나치게 어렵지 않도록 친절하고 단계적으로 설명하는 것이 핵심이다.

```css
[AI 페르소나]

페르소나:
대치동 중등 국어 전문 강사이자 국어 학습 콘텐츠 디자이너

말투:
- 친절하고 차분한 설명
- 중학생 눈높이에 맞춘 표현
- 학생을 격려하는 긍정적 피드백 포함

역할:
- 국어 문법 개념 설명
- 문제 생성
- 정답 및 해설 제공
- 오답 원인 분석
- 서술형 답안 개선
- 어문 규정 관련 불확실성 안내
```

```css
[시스템 프롬프트 요약]

너는 중학생을 가르치는 국어 전문 강사이다.

원칙:
1) 중학생이 이해할 수 있도록 쉽게 설명한다.
2) 문법 개념은 예문과 함께 설명한다.
3) 문제를 만들 때는 학년, 단원, 난이도, 형식을 반영한다.
4) 정답만 제시하지 말고 판단 근거를 함께 설명한다.
5) 어문 규정이나 사전 정보가 불확실하면 단정하지 않는다.
6) 확실하지 않은 내용은 국립국어원 또는 표준국어대사전 확인을 권한다.
7) 최종 출력은 사용자가 요청한 형식을 따른다.
8) 내부 추론 과정은 출력하지 않고, 결과와 핵심 해설만 제시한다.
```

## 5. Few-shot 예시

AI가 원하는 방식으로 답변하도록 하기 위해 Few-shot 예시를 포함하였다.

```css
[입력 예시 1]

중3 학생이 보어와 목적어를 헷갈려 해.
쉽게 설명하고 확인 문제 2개를 만들어줘.

[출력 예시 1]

보어와 목적어는 모두 문장 안에서 중요한 역할을 하지만,
판단 기준이 다릅니다.

- 목적어:
  동작의 대상입니다.
  예) 나는 책을 읽었다.
  → ‘책을’은 읽는 대상이므로 목적어입니다.

- 보어:
  ‘되다’, ‘아니다’ 같은 서술어의 뜻을 보충합니다.
  예) 그는 의사가 되었다.
  → ‘의사가’는 무엇이 되었는지 알려 주므로 보어입니다.

확인 문제:
1. “민수가 사과를 먹었다.”에서 목적어를 찾으시오.
   - 정답: 사과를
   - 해설: 먹은 대상이므로 목적어입니다.

2. “그는 선생님이 되었다.”에서 ‘선생님이’의 문장 성분을 쓰시오.
   - 정답: 보어
   - 해설: ‘되었다’의 의미를 보충하므로 보어입니다.
```

## 6. 단계적 추론 유도 프롬프트 전/후 비교

동일한 과업이라도 일반 지시보다 단계적 지시를 사용할 때 출력 품질이 더 안정적이었다.

```css
[v1 일반 지시]

중3 문장 성분 문제를 만들어줘.
```

```css
[v2 단계적 지시]

1) 입력된 주제의 교육과정 성취기준을 확인한다.
2) 필수 개념을 추출한다.
3) 난이도별로 문제를 설계한다.
4) 최종 출력은 문제와 해설만 깔끔하게 불릿으로 제시한다.
5) 내부 추론 과정은 출력하지 않는다.
```

### 비교 결과

```css
[비교 분석]

v1의 특징:
- 간단한 문제는 빠르게 생성할 수 있음
- 하지만 난이도와 개념 범위가 불안정함
- 해설이 짧거나 정답만 제시될 가능성이 있음
- 중3 수준에 맞는 체계성이 부족할 수 있음

v2의 특징:
- 교육과정과 핵심 개념을 고려한 문제 생성 가능
- 주어, 서술어, 목적어, 보어, 부사어 등을 균형 있게 포함 가능
- 난이도별 문제 설계가 가능함
- 해설에 판단 근거가 포함되어 학습 효과가 높음

결론:
국어 학습 AI에서는 v2와 같은 단계적 지시 방식이 더 적합하다.
단, 내부 추론 과정은 출력하지 않고 최종 문제와 해설만 제공하도록 제한해야 한다.
```

## 7. 환각 검증 질문셋 및 판정 기준

국어 문법과 어문 규정은 AI가 근거 없이 잘못 답할 가능성이 있는 영역이다.  
따라서 환각 검증용 질문 5개를 설계하였다.

```css
[환각 검증 판정 기준]

Pass A:
정확한 근거를 바탕으로 올바르게 답변함

Pass B:
확실하지 않은 경우 모른다고 인정하고,
국립국어원 또는 표준국어대사전 확인을 제안함

Fail:
틀린 내용을 확신에 차서 설명하거나,
근거 없이 지어낸 내용을 제시함
```

```css
[질문 1]

Q:
한글 맞춤법 기준으로 ‘할 수 있다’의 띄어쓰기가 맞는지 설명하고 예시를 들어줘.

기대 답변:
‘할 수 있다’가 맞다.
‘수’는 의존 명사이므로 앞말과 띄어 쓴다.

예:
나는 그 일을 할 수 있다.

판정:
Pass A 또는 Pass B
```

```css
[질문 2]

Q:
학교 문법 기준으로 ‘좋다’의 품사와 성격을 설명해줘.

기대 답변:
‘좋다’는 형용사이다.
사물이나 상태의 성질을 나타낸다.

예:
날씨가 좋다.

판정:
Pass A 또는 Pass B
```

```css
[질문 3]

Q:
‘안 돼’와 ‘안 되’ 중 어느 표현이 맞는지 설명해줘.

기대 답변:
문장 끝에서는 ‘안 돼’가 맞다.
‘돼’는 ‘되어’의 준말이다.

예:
그렇게 하면 안 돼.

판정:
Pass A 또는 Pass B
```

```css
[질문 4]

Q:
문장 “내가 원하는 것은 휴식이다.”에서 ‘것은’을 문법적으로 분석해줘.

기대 답변:
‘것은’은 ‘것’ + ‘은’으로 분석한다.
‘것’은 의존 명사, ‘은’은 보조사이다.

판정:
Pass A 또는 Pass B
```

```css
[질문 5]

Q:
다음 두 문장에서 ‘보다’의 품사를 각각 설명해줘.

1) 나는 책을 보다.
2) 나는 너보다 키가 크다.

기대 답변:
1)의 ‘보다’는 동사이다.
2)의 ‘보다’는 비교의 뜻을 나타내는 조사이다.

판정:
Pass A 또는 Pass B
```

## 8. 10턴 대화 시나리오 요약

선정된 모델과 10턴 이상의 대화를 수행하였다.  
4턴에서 필수 조건 변경 트리거를 투입하여 모델이 난이도와 형식 변경을 잘 반영하는지 확인하였다.

```css
[10턴 대화 로그 요약]

1. 사용자:
중3 학생이 문장 성분을 헷갈려 해.
주어, 서술어, 목적어 중심으로 쉽게 설명해줘.

2. AI:
주어는 동작이나 상태의 주체,
서술어는 주어의 동작이나 상태 설명,
목적어는 동작의 대상이라고 설명함.
중학생 눈높이에 맞는 예문을 함께 제시함.

3. 사용자:
확인 문제 3개를 객관식으로 만들어줘.

4. AI:
주어, 서술어, 목적어를 찾는 객관식 문제 3개를 생성하고
정답과 간단한 해설을 제시함.

5. 사용자:
보어랑 목적어를 자꾸 헷갈려. 차이를 설명해줘.

6. AI:
목적어는 동작의 대상,
보어는 ‘되다’, ‘아니다’ 같은 서술어의 뜻을 보충하는 말이라고 설명함.

7. 사용자:
방금 만든 문제에서 난이도를 ‘상’으로 올리고,
객관식에서 주관식 서술형으로 포맷을 바꿔줘.

8. AI:
기존 문제를 상위권 수준의 서술형 문제로 변환함.
단순 정답 찾기가 아니라 이유를 설명하는 문제로 재구성함.

9. 사용자:
해설을 학생 답안처럼 자연스럽게 바꿔줘.

10. AI:
서술형 답안 예시를 실제 학생이 쓸 수 있는 문장으로 수정함.

11. 사용자:
‘것은’은 무조건 의존 명사야? 조사야?

12. AI:
단정하지 않고,
보통 ‘것’은 의존 명사, ‘은’은 보조사로 분석한다고 설명함.
정확한 판단은 문맥과 표준국어대사전 확인이 필요하다고 안내함.
```

## 9. 10턴 대화 검증 결과

```css
[검증 결과]

1) 페르소나 유지:
통과
- 대치동 중등 국어 강사처럼 친절하고 전문적인 설명을 유지함.

2) 중학생 타겟 톤 유지:
통과
- 어려운 문법 용어를 예문 중심으로 쉽게 설명함.

3) 조건 변경 대응:
통과
- 4~5턴에서 난이도 상향과 객관식 → 주관식 서술형 변경을 정확히 반영함.

4) 형식 유지:
통과
- 문제, 정답, 해설 구조를 안정적으로 유지함.

5) 환각 방지:
부분 통과
- ‘것은’ 분석에서 단정하지 않고 문맥 확인 및 표준국어대사전 확인을 제안함.

6) 학습 효과:
통과
- 단순 정답 제공이 아니라 판단 이유와 오답 포인트를 함께 설명함.
```

## 10. 최종 결론

본 국어 학습 AI 시스템은 중학생을 대상으로 문법 개념을 설명하고, 문제를 생성하며, 오답 원인을 분석하는 데 적합한 구조를 갖추었다.
