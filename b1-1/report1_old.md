

# [최종본] 국어 학습 AI 시스템 개발 및 검증 보고서

## 1. 프로젝트 개요 및 과업 정의
본 프로젝트는 중학생을 대상으로 국어 문법(문장 성분, 품사, 어문 규정 등) 학습을 돕는 AI 시스템을 설계하고 검증하는 것을 목표로 합니다.

*   **과업 명칭:** 중학생 대상 국어 문법 학습 콘텐츠 생성 및 질의응답
*   **타겟 사용자:** 중학생 (문법 개념 이해가 부족하거나 내신 대비가 필요한 학생)
*   **핵심 기능:** 
    *   난이도별 맞춤형 문법 개념 설명 및 문제 생성
    *   객관식/주관식/서술형 변환 및 첨삭
    *   어문 규정 관련 환각(Hallucination) 방지 및 오답 분석

---

## 2. 모델 비교 및 선정 결과
다양한 모델 구조를 비교한 결과, 정형 데이터 분류에 강한 **Random Forest**와 자연어 생성 능력이 뛰어난 **LLM**을 결합한 하이브리드 구조를 최종 선정하였습니다.

| 평가 축 | Random Forest + LLM 결합 (선정) | 단일 LLM 모델 | 단일 분류 모델 |
| :--- | :--- | :--- | :--- |
| **정확성 (분류)** | **0.90 (최상)** | 0.78 (중) | 0.88 (상) |
| **자연스러움** | **우수 (중등 눈높이)** | 우수 | 미흡 (텍스트 생성 불가) |
| **난이도 조절** | **우수 (안정적 반영)** | 보통 (난이도 이탈 발생) | 불가 |
| **환각 방지** | **우수 (방어적 답변 가능)** | 보통 (가짜 문법 생성) | 해당 없음 |

*   **선정 근거:** Random Forest로 학습자의 수준(학년, 오답 유형 등)을 정확히 판정한 뒤, 이를 LLM의 프롬프트에 주입함으로써 교육 콘텐츠의 **안정성**과 **유연성**을 동시에 확보함.

---

## 3. 입력 템플릿 및 프롬프트 구조화

### 3.1 국어 학습 과업 입력 템플릿
AI가 학습자의 요구를 정확히 파악하도록 9개 항목으로 구성하였습니다.

```markdown
[국어 학습 과업 입력 템플릿]
- 학년: (예: 중3)
- 단원: (예: 문장 성분)
- 세부 주제: (예: 주어, 서술어, 목적어, 보어 구분)
- 학습자 수준: (예: 기초 / 중간 / 상위권)
- 원하는 결과: (예: 개념 설명 / 문제 생성 / 오답 정리 / 서술형 답안 첨삭)
- 문제 형식: (예: 객관식 / 단답형 / 서술형 / 혼합형)
- 난이도: (예: 하 / 중 / 상)
- 톤: (예: 친절한 중등 국어 선생님 말투)
- 주의사항: (예: 어려운 용어는 쉽게 풀어서 설명하고, 모르는 규정은 단정하지 말 것)
```

### 3.2 시스템 프롬프트 (System Prompt)
**페르소나:** 대치동 중등 국어 전문 강사 및 콘텐츠 디자이너

1.  **쉬운 설명:** 중학생 수준에 맞춰 어려운 용어를 풀어서 설명한다.
2.  **예시 중심:** 모든 문법 개념은 반드시 구체적인 예문과 함께 제시한다.
3.  **근거 제시:** 정답뿐만 아니라 오답의 이유와 판단 근거를 상세히 설명한다.
4.  **환각 방지:** 불확실한 규정은 단정하지 않고 국립국어원 확인을 권고한다.
5.  **형식 준수:** 내부 추론 과정은 생략하고 최종 결과만 불릿(Bullet) 형태로 출력한다.

## 4. 프롬프트 개선 및 Few-shot 예시

### 4.1 프롬프트 버전 비교 (v1 → v2)
*   **v1 (일반 지시):** "중3 문장 성분 문제를 만들어줘." -> 난이도와 해설 품질이 불안정함.
*   **v2 (단계적 지시):** 교육과정 성취기준 확인 → 필수 개념 추출 → 난이도별 문제 설계 → 최종 출력 포맷팅 순으로 지시하여 **학습 효과와 일관성을 극대화**함.

### 4.2 Few-shot 예시 (3종)
1.  **정상 흐름:** 보어와 목적어의 차이를 예문(책을 읽었다 vs 의사가 되었다)으로 설명하고 확인 문제 생성.
2.  **되묻기 흐름:** 정보가 부족할 경우(예: "문제 만들어줘") 학년, 단원, 난이도를 다시 묻는 가이드 제공.
3.  **첨삭 흐름:** 서술형 문제에 대해 학생의 답안을 분석하고 모범 답안과 이유를 함께 제시.

## 5. 환각 검증 및 10턴 대화 결과

### 5.1 환각 검증 질문셋 (Pass A: 정확 / Pass B: 방어적 성공 / Fail: 오류)
*   **Q1 (띄어쓰기):** '할 수 있다' 분석 -> **Pass A** (의존 명사 띄어쓰기 준수)
*   **Q2 (품사):** '좋다'의 품사 -> **Pass A** (형용사로 정확히 분류)
*   **Q3 (맞춤법):** '안 돼' vs '안 되' -> **Pass A** ('돼'는 '되어'의 준말임을 명시)
*   **Q4 (문장 분석):** '것은'의 분석 -> **Pass A** (의존 명사 + 보조사)
*   **Q5 (다의어):** '보다'의 품사 구분 -> **Pass A** (동사와 조사로 정확히 구분)

### 5.2 10턴 대화 시나리오 검증
*   **검증 내용:** 1~6턴까지 개념 설명 후, **7턴에서 난이도 상향 및 서술형으로 포맷 변경** 요청.
*   **결과:** AI는 기존 맥락을 유지하며 상위권 수준의 서술형 문항으로 완벽히 전환함. 10턴 이후에도 페르소나와 톤을 안정적으로 유지함.

## 6. 심층 면접 답변 (핵심 요약)

*   **환각 정의:** 국어 문법은 고정된 사실(Fact) 기반이므로, 지어낸 정보는 '창의성'이 아닌 **'사실 오류'**로 엄격히 규정함.
*   **모델 선정 이유:** 난이도 조절의 **일관성**을 위해 Random Forest(분류)와 LLM(생성)의 하이브리드 구조가 최적임.
*   **비용 최적화:** 무료 모델 사용 시, 태스크를 원자 단위로 쪼개는 **'체인 오브 노트(Chain of Notes)'** 기법 적용.
*   **정책 변동 대응:** 최신 어문 규정 반영을 위해 **RAG(검색 증강 생성)** 파이프라인을 구축하여 실시간 데이터를 참조함.
*   **장기 대화 유지:** 10턴 이상 대화 시 발생하는 맥락 이탈을 막기 위해 **'요약 메모리 버퍼'**를 도입하여 핵심 설정을 재주입함.
