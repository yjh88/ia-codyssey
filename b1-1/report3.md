# 문서 1. AI 모델 비교 및 선정

## 1. 평가 목적

이번 과제의 목적은 중학교 3학년 학생이 국어 문법을 학습할 때 도움을 줄 수 있는 AI 튜터를 설계하는 것이다.

AI 튜터는 단순히 정답만 알려주는 것이 아니라, 학생이 문법 개념을 이해하고 스스로 문제를 풀 수 있도록 설명해야 한다. 따라서 여러 AI 모델을 비교하여 국어 문법 학습에 가장 적합한 모델을 선정하였다.

비교 대상 모델은 다음과 같다.

- GPT-4o
- Claude
- Gemini

---

## 2. 모델 비교 기준

AI 모델을 비교하기 위해 다음 7가지 기준을 사용하였다.

| 평가 기준 | 설명 |
|---|---|
| 개념 설명 능력 | 문법 개념을 학생 수준에 맞게 쉽게 설명하는 능력 |
| 문제 생성 능력 | 학습 내용에 맞는 객관식, 단답형, 서술형 문제를 만드는 능력 |
| 오답 피드백 능력 | 학생의 오답을 분석하고 이유를 설명하는 능력 |
| 대화 흐름 유지 능력 | 이전 대화 내용을 기억하고 이어서 설명하는 능력 |
| 환각 가능성 | 사실과 다른 내용을 만들어낼 가능성 |
| 학습 친화성 | 친절하고 격려하는 말투로 학습을 도와주는 정도 |
| 사용 편의성 | 실제 과제 수행과 학습에 사용하기 쉬운 정도 |

---

## 3. 모델별 평가표

| 평가 기준 | GPT-4o | Claude | Gemini |
|---|---|---|---|
| 개념 설명 능력 | 매우 우수. 중학생 수준에 맞춰 단계적으로 설명함 | 우수. 차분하고 긴 설명에 강함 | 보통 이상. 핵심 설명은 가능하나 세부 설명은 부족할 때가 있음 |
| 문제 생성 능력 | 매우 우수. 난이도 조절과 다양한 유형 생성 가능 | 우수. 논리적인 문제 생성에 강함 | 보통. 간단한 문제 생성은 가능하지만 정교함은 다소 부족함 |
| 오답 피드백 능력 | 매우 우수. 왜 틀렸는지 구체적으로 설명 가능 | 우수. 친절하지만 설명이 길어질 수 있음 | 보통. 간단한 피드백은 가능하나 깊이 있는 분석은 약할 수 있음 |
| 대화 흐름 유지 능력 | 우수. 이전 문맥을 잘 반영함 | 매우 우수. 긴 대화 흐름 유지에 강함 | 보통. 짧은 대화에서는 무난하나 긴 흐름에서는 약할 수 있음 |
| 환각 가능성 | 비교적 낮음. 단, 검증 절차 필요 | 낮은 편. 신중한 답변을 하는 경향 | 보통. 세부 문법 설명에서 오류 가능성 있음 |
| 학습 친화성 | 매우 우수. 학생에게 친절하고 격려하는 말투 가능 | 우수. 안정적이고 차분한 말투 | 보통 이상. 친절하지만 맞춤형 튜터 느낌은 약할 수 있음 |
| 사용 편의성 | 매우 우수. 다양한 형식의 결과물을 쉽게 생성 | 우수. 글 정리와 설명에 강함 | 우수. 접근성과 속도 면에서 장점 |

## 4. 모델별 장단점 분석

### 4.1 GPT-4o

#### 장점
- 학생 수준에 맞춘 설명이 자연스럽다.
- 문법 개념을 예시와 함께 쉽게 설명할 수 있다.
- 문제 생성, 채점, 오답 피드백이 모두 가능하다.
- 대화형 튜터 역할에 적합하다.
- 말투를 친절한 강사처럼 조절할 수 있다.

#### 단점
- 문법 용어를 설명할 때 가끔 지나치게 단순화할 수 있다.
- 모든 답변이 항상 정확한 것은 아니므로 검증 과정이 필요하다.
- 사용 환경에 따라 응답 속도가 달라질 수 있다.

---

### 4.2 Claude

#### 장점
- 긴 글을 안정적으로 처리하는 능력이 뛰어나다.
- 설명이 차분하고 논리적이다.
- 학생의 답변을 분석하여 부드럽게 피드백하는 데 강점이 있다.
- 안전하고 신중한 답변을 하는 편이다.

#### 단점
- 설명이 길어져 중학생에게 부담이 될 수 있다.
- 문제를 짧고 간단하게 만드는 데에는 조정이 필요할 수 있다.
- 한국어 국어 문법의 세부 개념에서 추가 검토가 필요하다.

---

### 4.3 Gemini

#### 장점
- 빠른 응답과 접근성이 좋다.
- 간단한 개념 설명과 문제 생성이 가능하다.
- 짧은 학습 활동에 활용하기 좋다.
- 다양한 구글 서비스와의 연동 가능성이 있다.

#### 단점
- 복잡한 문법 설명에서는 세부 정확도가 떨어질 수 있다.
- 오답 원인 분석이 다소 단순할 수 있다.
- 학습자 수준에 맞춘 맞춤형 피드백은 다른 모델보다 약할 수 있다.

## 5. 최종 선정 모델

최종적으로 이번 과제에서는 **GPT-4o**를 선택하였다.

### 선정 이유

GPT-4o는 중학교 3학년 학생에게 국어 문법을 설명하는 데 가장 적합하다고 판단하였다. 특히 다음과 같은 점에서 강점이 있었다.

1. 문법 개념을 쉽게 설명할 수 있다.
2. 학생 수준에 맞게 예시를 조절할 수 있다.
3. 객관식, 단답형, 서술형 문제를 만들 수 있다.
4. 학생의 오답에 대해 친절한 피드백을 제공할 수 있다.
5. 대화형 학습 튜터 역할을 자연스럽게 수행할 수 있다.
6. 페르소나를 설정하면 일정한 말투와 학습 방식을 유지할 수 있다.

따라서 국어 문법 학습용 AI 튜터인 **나라 강사**를 설계하기 위한 기본 모델로 GPT-4o를 사용하였다.

## 7. 모델 비교 결론

세 모델 모두 학습 보조 도구로 활용할 수 있지만, 이번 과제의 목적에는 GPT-4o가 가장 적합하였다.

Claude는 긴 설명과 안정적인 글 구성에 강점이 있었고, Gemini는 접근성과 속도 면에서 장점이 있었다. 그러나 중학생 학습자를 대상으로 한 맞춤형 설명, 문제 생성, 오답 피드백, 대화형 학습 흐름 유지까지 고려하면 GPT-4o가 가장 균형 잡힌 성능을 보였다.

따라서 본 보고서에서는 GPT-4o를 기반으로 국어 문법 학습용 AI 튜터 시스템을 설계하였다.

# 문서 2. AI 튜터 시스템 설계

## 1. 시스템 설계 목적

이 시스템은 중학교 3학년 학생이 국어 문법 중 문장 성분을 쉽게 이해하고 연습할 수 있도록 돕기 위해 설계되었다.

특히 학생이 어려워하는 다음 내용을 중심으로 학습을 진행한다.

- 주어와 서술어의 개념
- 목적어와 보어의 차이
- 문장 성분 찾기
- 조건에 맞는 문장 만들기
- 오답 원인 분석
- 단계별 문제 풀이

AI 튜터는 학생에게 정답을 바로 알려주기보다, 스스로 생각할 수 있도록 힌트와 설명을 제공한다.

## 2. AI 튜터 페르소나

### 튜터 이름
나라 강사

### 역할
나라 강사는 중학교 3학년 국어 문법을 친절하게 설명해 주는 AI 국어 튜터이다.

### 말투
- 친절하고 부드러운 말투를 사용한다.
- 학생을 비난하지 않는다.
- 틀린 답을 말해도 긍정적으로 반응한다.
- 어려운 문법 용어는 쉬운 말로 풀어서 설명한다.
- 필요할 때는 예문을 들어 설명한다.

### 학습 태도
- 학생이 스스로 답을 찾을 수 있도록 유도한다.
- 정답보다 이해 과정을 중요하게 생각한다.
- 오답이 나오면 왜 틀렸는지 차근차근 설명한다.
- 학생의 수준에 맞춰 난이도를 조절한다.
- 학습이 끝날 때 간단한 정리와 격려를 제공한다.

---

## 3. 페르소나 프롬프트

다음은 나라 강사를 설정하기 위한 페르소나 프롬프트이다.

```text
너는 중학교 3학년 학생을 가르치는 친절한 국어 문법 튜터 '나라 강사'야.

너의 역할은 학생이 국어 문법, 특히 문장 성분을 쉽게 이해하도록 돕는 것이야.

다음 규칙을 반드시 지켜.

1. 항상 한국어로 대답해.
2. 중학교 3학년 수준에 맞게 설명해.
3. 어려운 문법 용어는 쉬운 말로 풀어 설명해.
4. 예문을 사용해서 설명해.
5. 학생이 틀린 답을 해도 비난하지 말고 부드럽게 고쳐 줘.
6. 정답만 말하지 말고 왜 그런지 이유를 설명해.
7. 필요하면 힌트를 먼저 주고, 학생이 생각할 시간을 줄 수 있게 해.
8. 문제를 낼 때는 난이도를 조절해.
9. 한 번에 너무 많은 내용을 설명하지 말고 단계적으로 알려 줘.
10. 답변 마지막에는 짧은 격려 문장을 넣어 줘.

학습 주제는 다음과 같아.

- 주어
- 서술어
- 목적어
- 보어
- 문장 성분 구분
- 조건에 맞는 문장 만들기

학생의 답변을 분석하고, 이해 정도에 따라 설명, 문제, 힌트, 피드백을 제공해.
```

## 4. 입력 템플릿 설계

AI 튜터가 일관성 있게 답변하도록 하기 위해 입력 템플릿을 만들었다.

---

## 4.1 입력 템플릿 v1

```text
나는 중학교 3학년 학생이야.
국어 문법을 공부하고 있어.
문장 성분에 대해 쉽게 설명해 줘.
문제를 내 주고, 내가 답하면 피드백해 줘.
```

---

## 4.2 입력 템플릿 v1의 문제점

입력 템플릿 v1은 간단하고 사용하기 쉽지만, 다음과 같은 한계가 있다.

1. 학습 주제가 넓어서 답변이 너무 일반적으로 나올 수 있다.
2. 학생의 수준이나 현재 이해도가 반영되지 않는다.
3. 문제 유형이 정해져 있지 않다.
4. 피드백 방식이 구체적으로 설정되어 있지 않다.
5. AI가 어떤 순서로 수업을 진행해야 하는지 명확하지 않다.

따라서 더 구체적인 입력 템플릿이 필요하다.

---

## 4.3 입력 템플릿 v2

```text
너는 중학교 3학년 국어 문법 튜터 '나라 강사'야.

나는 문장 성분을 공부하는 학생이야.
오늘 학습할 내용은 [학습 주제]야.

내 수준은 [초급/중급/심화] 중에서 [수준]이야.
먼저 핵심 개념을 쉽게 설명해 줘.
그다음 예문 2개를 들어 줘.
이후 [문제 유형] 문제를 [문제 개수]개 내 줘.

내가 답을 하면 다음 기준으로 피드백해 줘.

1. 정답 여부
2. 왜 맞았는지 또는 왜 틀렸는지
3. 헷갈리기 쉬운 점
4. 다시 풀어 볼 수 있는 비슷한 문제 1개

답변은 중학교 3학년 학생이 이해할 수 있게 해 줘.
너무 길게 설명하지 말고 단계적으로 설명해 줘.
마지막에는 짧은 격려 문장을 넣어 줘.
```

## 5. 입력 템플릿 v1과 v2 비교

| 구분 | v1 | v2 |

| 구체성 | 낮음 | 높음 |
| 학습 주제 설정 | 문장 성분 전체로 넓음 | 학습 주제를 직접 지정 가능 |
| 학생 수준 반영 | 없음 | 초급, 중급, 심화로 설정 가능 |
| 문제 유형 지정 | 없음 | 객관식, 단답형, 서술형 등 지정 가능 |
| 문제 개수 지정 | 없음 | 원하는 개수 지정 가능 |
| 피드백 기준 | 단순히 피드백 요청 | 정답 여부, 이유, 헷갈리는 점, 추가 문제 포함 |
| 학습 흐름 | 불명확 | 개념 설명 → 예문 → 문제 → 피드백 순서 |
| 튜터 말투 | 명확하지 않음 | 나라 강사 페르소나 유지 가능 |

---

## 6. 입력 템플릿 개선 근거

v2 템플릿은 v1보다 학습 목적에 더 적합하다.

그 이유는 다음과 같다.

첫째, 학습 주제를 구체적으로 설정할 수 있다.  
예를 들어 “문장 성분”이라고만 입력하면 범위가 넓지만, “목적어와 보어의 차이”라고 입력하면 더 정확한 설명을 받을 수 있다.

둘째, 학생의 수준을 반영할 수 있다.  
초급 학생에게는 쉬운 예문과 간단한 설명이 필요하고, 심화 학생에게는 헷갈리는 문장이나 예외적인 경우까지 설명할 수 있어야 한다.

셋째, 문제 유형과 개수를 지정할 수 있다.  
객관식 문제, 단답형 문제, 서술형 문제 등 학습 상황에 맞는 문제를 만들 수 있다.

넷째, 피드백 방식이 명확하다.  
단순히 “틀렸어”라고 말하는 것이 아니라, 왜 틀렸는지와 어떤 점을 조심해야 하는지를 알려 줄 수 있다.

다섯째, 학습 흐름이 안정적이다.  
개념 설명, 예문, 문제 풀이, 피드백 순서로 진행되므로 실제 수업처럼 활용하기 좋다.

이를 통해 학생이 왜 틀렸는지 이해하고 다시 풀 수 있도록 하였다.

## 4. 재도전 및 정답 확인 단계

Turn 5에서 학생은 새 문제를 정확히 풀었다.  
AI 튜터는 정답 여부를 알려 주고, 왜 맞았는지 간단히 설명하였다.

이 단계는 학생이 이전 오답을 수정하고 개념을 다시 적용했다는 점에서 의미가 있다.

---

## 5. 심화 개념 연결 단계

Turn 6부터는 목적어와 보어의 차이를 다루었다.  
학생들이 가장 많이 헷갈리는 부분이므로 다음 기준을 제시하였다.

| 구분 | 판단 기준 | 예시 |
|---|---|---|
| 목적어 | 동작의 대상 | 책을 읽었다 |
| 보어 | 서술어의 의미를 완성 | 학생이 되었다 |

특히 보어는 중학교 문법에서 주로 **되다**, **아니다**와 관련된다는 점을 강조하였다.

---

## 6. 보어 오답 피드백 단계

Turn 7에서 학생은 “의사가”를 목적어라고 잘못 판단하였다.  
AI 튜터는 이를 오답으로 판단하고, 반드시 오답 해설을 제공하였다.

오답 원인은 다음과 같다.

- ‘의사가’는 동작의 대상이 아니다.
- ‘되었다’의 의미를 완성해 주는 말이다.
- 따라서 목적어가 아니라 보어이다.

이 피드백은 Few-shot 예시에서 설정한 오답 피드백 규칙을 따른 것이다.

---

## 7. 조사 기준 오류 교정 단계

Turn 9에서는 학생이 “이/가가 붙으면 주어가 아닌가요?”라고 질문하였다.  
AI 튜터는 조사만으로 문장 성분을 판단하면 안 된다고 설명하였다.

예시 비교:

```text
학생이 웃었다.
나는 학생이 되었다.
```

첫 번째 문장의 ‘학생이’는 주어이고, 두 번째 문장의 ‘학생이’는 보어이다.  
이를 통해 같은 형태라도 문장에서 맡은 역할에 따라 문장 성분이 달라진다는 점을 설명하였다.
